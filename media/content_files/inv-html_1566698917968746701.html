1 - CSS Preprocessor

CSS Preprocessor

Extending CSS to Provide Extra Functionality

Advantages to using a preprocessor

  • Cleaner code with reusable pieces and variables
  • Saves you time
  • Easier to maintain code with snippets and libraries
  • Calculations and logic
  • More organized and easy to setup

CSS Preprocessors:

  • Sass - Most Popular one
  • Less
  • Stylus

Features:

  • VARIABLES
  • NESTING
  • PARTIALS
  • IMPORTS
  • MIXINS
  • EXTENDS/INHERITANCE
  • OPERATORS

1.1 Variables:


  $darkBrown: #332d29;
  $grey: #666;
  $baseFontFamily: 'Open Sans', helvetica, sans-serif;
  
  body {
      background: $grey;
      font-family: $baseFontFamily;
  }
  footer {
      background: $darkBrown;
      color: $grey;
  }

1.2 Nesting:


  section .container { 
    background: #333;
    #home {
        padding: 10 12px;
        div {
            h1 {
                background: whiteSmoke;
            }
            span, p {
                color: #333;
            }
        }
    }
  }

1.3 Partials:

In CSS if we separate the files, we get tons of http requests, otherwise we would have a extremely long file.

More requests = Less speed

1.3 Partials:

With partials we can separate stylesheet as we want without having to worry about those http request.

The output will be a single css file.

1.4 Imports

We can separate .scss files and import.


  @import "compass/css3";
  @import "susy";
  @import "animation/animate";
  @import "myfile.scss"

8 - Custom attributes

data-*

Why?

to store custom data to the page or application

Have you stored data inside class or id attribute so you can use in javascript?

[{ id:5, name:"Seytech" }]
  
  <div id="5"> Seytech </div>

We could use like this?

[{ id:5, name:"Seytech", category: 'education' }]
  
  <div data-id="5" data-category="education"> Seytech </div>

data-*

The data attribute name must be at least one character long and must be prefixed with 'data-'. It should not contain any uppercase letters.

How to use from js?

elem.hasAttribute(name) – checks for existence.
  elem.getAttribute(name) – gets the value.
  elem.setAttribute(name, value) – sets the value.
  elem.removeAttribute(name) – removes the attribute.

9 - Block vs Inline Elements

A block-level element

always starts on a new line and takes up the full width available

An inline-level element

An inline element does not start on a new line and only takes up as much width as necessary.

Block elements

<div>, <h1-h6>, <header>, <footer>, <ul>, <li>, <ol>, <table> etc

Inline elements

<span>, <a>, <button>, <img>, <input>, <strong>, <script> etc

Div vs Span

  • <div> usually used as a container for other HTML elements and to separate them for the rest.
  • <span> usually used as a container for HTML text. But essentially, it is used to style a certain text within a larger text element.
